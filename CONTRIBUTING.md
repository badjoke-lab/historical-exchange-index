# Contributing

Thanks for your interest in Historical Exchange Index (HEI).

HEI is a registry-first project focused on exchange history, record structure, and source-backed classification.
At this stage, reviewability matters more than volume.

## Before opening a PR

For most changes, please open an Issue first.

This is especially important for:

- data corrections
- new record proposals
- status or death-reason disputes
- schema changes
- methodology changes
- large batch additions

## What helps most

The most useful contributions are:

- clear bug reports
- exchange record corrections
- stronger source links
- archive links
- small, reviewable documentation improvements
- narrowly scoped code fixes

## Data-related contributions

If you want to suggest a data correction or missing record, include as much of the following as you can:

- exchange name
- page URL on HEI, if one exists
- what looks wrong
- what you think should be changed
- supporting source links
- archive links when relevant

Please prefer source-backed corrections over unsourced claims.

## PR expectations

Good PRs for this repository are:

- small
- specific
- easy to review
- aligned with existing structure
- supported by sources when touching records or methodology

Please avoid:

- very large bulk data dumps
- unsourced mass additions
- schema rewrites without prior discussion
- mixed PRs that combine unrelated fixes

## Methodology and edge cases

HEI is still evolving.
Some classifications involve edge cases, competing sources, or incomplete records.

That means a technically valid PR may still need discussion if it affects:

- what counts as an exchange
- how dead / merged / acquired / rebranded states are handled
- how evidence quality is judged
- how archived URLs should be prioritized

## Style

When possible:

- keep wording precise and neutral
- avoid hype or promotional phrasing
- favor clarity over verbosity
- preserve the project’s archival / registry tone
